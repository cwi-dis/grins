GRiNS Editor for SMIL 1.0 Version 1.5.1 Release Notes for Windows

These are the release notes for the GRiNS Editor and Player for SMIL 1.0 Version 1.5.1 for Windows.

Changes since Version 1.0

There have been too many changes since 1.0 to list them all.  Some of the more important ones are listed below.

The user interface for part of the Editor has changed considerably. The user interface is now in line with GRiNS 1.5 for G2.

  • Time in the Structure View is now left-to-right instead of top-to-bottom, to align with the time flow in the Timeline View.
  • Property editing dialogs are completely overhauled.
There is now a command to upload the finished document plus data objects to a web server.

The GRiNS Tutorials are now included in the GRiNS installer.

GRiNS now saves the document in a file with the .grins extensions instead of the .smil extension. This has the advantage that GRiNS can coexist with other players for SMIL documents.

Start and end times on anchors are now implemented.

Windows User Interface Open Issues

GRiNS can render HTML either through the Internet Explorer HTML control (part of IE4) or the built-in WebsterPro HTML control. This is user-selectable throught the Preferences dialog. In some circumstances the IE control causes the Player/Editor to crash. If this happens, use the WebsterPro control.

Each control has its good and bad points. IE4 has more features but always shows scrollbars which is not very pretty. Webster is a HTML 3 control only, and has problems with some things like using tags in URLs and accessing files via network neighborhood naming. Both controls can be slow rendering the data on the screen.

If sounds or videos are not played correctly on your machine (e.g. *.aiff or *.mpeg) try installing Active Movie, also sometimes known as Media Player. It is included in Service Pack 4 for NT, and for Windows 95 you should see Microsoft's Media Player download page for details.

Open Issues

Using the Edit Content command from the Edit menu starts the program registered for that file type. This may not always be an Editor.

If text does not fit in its region, a small red box is shown in the bottom right corner of the preview player to indicate that not all text is shown.

The SMIL standard does not permit a jump (hyperlink) into a switch node. GRiNS currently does not prevent the user from creating such hyperlinks and also does not warn about them.

Windows NT does not allow playback of two separate audio channels simultaneously. Windows 98 does not have this problem.

Windows that were hidden behind other windows may pop up at seemingly random moments. Move your windows so that they don't fully overlap. In that way you can easily get back to the window you were working with.

GRiNS can crash when invalid values are entered for the Clip begin or Clip end properties in video nodes. Make sure that the values are valid (or empty) before trying to play back a document using Clip begin or Clip end.

Partial anchors on video nodes can be created and are saved correctly, but during playback in GRiNS, the whole video region is sensitive instead of just the anchor area.

Unsupported SMIL Features

There are only a few SMIL features that are not supported by the GRiNS Editor and Player for SMIL. Documents that make use of these features are parsed correctly, but the features are ignored.
  • fit="fill" and fit="scroll" attribute values in region element.
  • name="pics-label" attribute value in meta element.
  • alt attribute in media object elements.
  • fill attribute.
