GRiNS Player 2.1 for SMIL 2.0, Release Notes for Macintosh

These are the release notes for GRiNS Player 2.1 for SMIL 2.0, Release Notes for Macintosh, build 30. A more recent version of these release notes, as well as more recent builds of the player, may be available through http://www.oratrix.com/GRiNS/SMIL-2.0/.

This release should be viewed as a maintainance release. The reason for the jump in version number is to keep version numbers consistent across the Oratrix GRiNS product family: the core SMIL execution engine is the same in GRiNS Player 2.1, GRiNS Editor 2.1 for RealOne and GRiNS Editor 2.1 for SMIL 2.0.

The player can play back RealMedia format files (RealVideo, RealAudio, RealPix, RealText) if the RealPlayer is installed.

The player can always playback a subset of QuickTime media items. If you have QuickTime for Windows installed the player will detect this and play back a richer set of QuickTime media items.

Mimetype handling is more precise in this release.

The SMIL execution engine has been brought up to spec with respect to the final SMIL 2.0 standard. Moreover, many minor bugs have been fixed, especially in the areas of pausing in excl nodes, hyperlinking and transitions.

Open Issues

  • Viewports (SMIL terminology, windows in Mac terminology) open overlapping, which is a bit of a nuisance.
  • If your SMIL document repeatedly opens and closes viewports you may end up without a player control panel. The Play menu still works, though.
  • Positioning MPEG movies (with clipBegin, for instance) is very slow.

SMIL 2.0 Compatibility

This player supports the SMIL 2.0 Language Profile completely. The SMIL 2.0 Language Profile consists of the following modules. Where appropriate, a note is added to clarify the level of support. For more information about SMIL 2.0 and the SMIL 2.0 Language Profile, see here.

  • Structure functionality
    • Structure Module
  • Metainformation functionality
    • Metainformation Module
  • Timing functionality
    • BasicInlineTiming Module
    • SyncbaseTiming Module
    • EventTiming Module
    • MinMaxTiming Module
    • RepeatTiming Module
    • RepeatValueTiming Module
    • WallclockTiming Module
    • MultiArcTiming Module
    • AccessKeyTiming Module
    • BasicTimeContainers Module
    • ExclTimeContainers Module
    • RestartTiming Module
    • SyncBehavior Module
      • Only syncBehavior="independent" is supported.
    • SyncBehaviorDefault Module
    • RestartDefault Module
    • FillDefault Module
    • MediaMarkerTiming Module
  • Animation functionality
    • BasicAnimation Module
      • The attribute attributeType is recognized but ignored (attributeType is always XML).
  • Content Control functionality
    • BasicContentControl Module
    • CustomTestAttributes Module
    • PrefetchControl Module
    • SkipContentControl Module
  • Layout functionality
    • BasicLayout Module
    • AudioLayout Module
    • MultiWindowLayout Module
    • HierarchicalLayout Module
  • Linking functionality
    • BasicLinking Module
    • LinkingAttributes Module
    • ObjectLinking Module
      • Fully supported, but only links in HTML documents can be overridden using the fragment attribute.
  • Transition Effects functionality
    • BasicTransitions Module
      • Type miscShapeWipe is recognized but not implemented.
      • Only some subtype values are implemented.
      • The direction attribute is recognized but ignored (direction is always forward).
    • TransitionModifiers Module
      • The TransitionModifiers attributes (borderColor, borderWidth, horzRepeat, and vertRepeat) are recognized but ignored.
  • Media functionality
    • BasicMedia Module
    • MediaClipping Module
    • MediaClipMarkers Module
    • MediaParam Module
      • The only param value that is recognized is name="fgcolor" value="some color specification". This value only works on media with the MIME type text/plain.
    • BrushMedia Module
    • MediaAccessibility Module
      • Fully supported in that the readIndex attribute is parsed correctly. There is no support for actually reading out the alt, title, and longdesc attributes.

In addition to the modules from the SMIL 2.0 Language Profiles, the player also supports the following SMIL 2.0 Modules.

  • Time Manipulations functionality
    • Time Manipulations Module
      • Only supported for animation elements (animate, animateColor, animateMotion, set, transitionFilter).
  • Animation functionality
    • SplineAnimation
  • Transition Effects functionality
    • InlineTransitions
