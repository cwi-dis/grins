GRiNS Player beta for SMIL 2.0, Release Notes for Windows

These are the release notes for GRiNS Player beta for SMIL 2.0, Release Notes for Windows, build 36, November 23, 2000. A more recent version of these release notes, as well as more recent builds of the player, may be available through http://www.oratrix.com/GRiNS/SMIL-2.0/.

Note that this is an early beta, and the playback engine has not been tested very thoroughly with large documents. You may experience instabilities.

Note: There has been a problem with hanging processes with Build 35 and some older versions of the Player on Windows. This problem was fixed in Build 36. Before uninstalling older versions of the Player, and before installing the Player on top of an older version, be sure to kill any processes with the name GRiNSP_SV2.exe (use the Task Manager to kill these processes).

Changes Since Build 35

  • The problem that the Player process didn't actually go away on exit was solved.

Changes Since Build 33

  • All attributes that are valid according to the current SMIL 2.0 Language Profile are now recognized, although not all attributes are implemented.
  • The SMIL 2.0 default namespace is now http://www.w3.org/2000/SMIL20/CR/Language older namespaces are recognized but result in a warning message.
  • The fill="hold" attribute value has been implemented.
  • The fill="transition" attribute value has been implemented.
  • A problem was fixed when the presentation was paused while there were children in an excl elements that were paused due to excl/priorityClass semantics.
  • A problem was fixed with container elements that need to be frozen from the start.
  • A problem was fixed when jumping to an element in a sequence where one of the elements before the target element had indefinite duration.
  • A problem was fixed with playing a video element in a region that didn't fit in it's parent region (and was therefore clipped to the size of the parent region).
  • A problem was fixed in the Preferences dialog. The check marks for the systemOverdubOrSubtitle and systemCaptions attributes were exchanged.
  • The override="uid-only" attribute value on customTest elements has been implemented.
  • The param element is now recognized.
  • The syntax for multiple values in the systemRequired and customTest attributes has been implemented.

Changes Since Build 32

  • There were occasional crashes for documents using the regionName attributes. Fixed.

Changes Since Build 29

  • Support for viewportOpenEvent and viewportCloseEvent.
  • Support for synchronization to begin/end/activateEvent on area and anchor element.
  • Added playback support for Windows Media Video and Audio files.
  • Various fixes to the scheduler.

Changes Since Build 28

  • All known problems with rendering RealVideo, RealText, and RealPix have been fixed.
  • The delay in responding to mouse events has been fixed.
  • A number of problems were fixed having to do with multiple layout elements inside a switch inside the head section.
  • URLs referencing source materials can now have queries in them (i.e., URLs of the form src="image.gif?bitrate=500") which will be ignored when playing back from the local disk. This makes the player more compatible with the RealPlayer which uses these queries to encode extra information

Changes Since Build 27

Some users have reported a problem with the speed with which the application responds to mouse events when a video is playing. The application does respond, but it may take up to several seconds before it does. Often restarting the application helps. We are investigating the problem and will try to come up with a solution.

  • RealVideo, RealText, and RealPix media elements now obey the z-index and fit attributes properly.
  • More fixes to the scheduler.

Changes Since Build 24

  • The actuate="onLoad" attribute value for anchors is now implemented.
  • When linking to a document that GRiNS cannot handle itself (i.e. all documents except SMIL documents), the document is handed off to the appropriate application.
  • RealVideo, RealText, and RealPix media elements can now be used as the source of hyperlinks.
  • Various improvements in the scheduler. In particular, an excl node with no resolved children and no explicit duration will now not play. Also, a node with 0 duration but with a min attribute greater than 0 will play and end after the min time is up.

Changes Since Build 23

  • Improvements to the hyperlinking code.
  • Support for embedded HTML documents (i.e., you can use HTML documents as media objects).
  • The readIndex attribute is now recognized.

SMIL 2.0 Compatibility

This player supports the SMIL 2.0 Language Profile, with the following notes:
  • Structure functionality
    • Structure Module
      • Fully supported
  • Metainformation functionality
    • Metainformation Module
      • Fully supported
  • Timing functionality
    • BasicInlineTiming Module
      • Fully supported
    • SyncbaseTiming Module
      • Fully supported
    • EventTiming Module
      • Fully supported
    • MinMaxTiming Module
      • Fully supported
    • WallclockTiming Module
      • Fully supported
    • MultiArcTiming Module
      • Fully supported
    • BasicTimeContainers Module
      • Fully supported
    • ExclTimeContainers Module
      • Fully supported
    • PrevTiming Module
      • Fully supported
    • RestartTiming Module
      • Fully supported
    • SyncBehavior Module
      • Only syncBehavior="independent" is supported
    • SyncBehaviorDefault Module
      • Fully supported
    • RestartDefault Module
      • Fully supported
    • FillDefault Module
      • Fully supported
  • Animation functionality
    • BasicAnimation Module
      • The target attribute soundLevel is recognized but not implemented.
      • The attribute attributeType is recognized but ignored (attributeType is always XML).
  • Content Control functionality
    • BasicContentControl Module
      • Fully supported
    • CustomTestAttributes Module
      • Fully supported
    • PrefetchControl Module
      • To be Supplied
    • SkipContentControl Module
      • To be Supplied
  • Layout functionality
    • BasicLayout Module
      • The scroll value for fit attribute is recognized but not implemented.
    • AudioLayout Module
      • The soundLevel attribute is recognized but ignored.
    • MultiWindowLayout Module
      • Fully supported
    • HierarchicalLayout Module
      • For media with no intrinsic sizes (html, text, ...), the registration point attributes are recognized but ignored
  • Linking functionality
    • BasicLinking Module
      • Fully supported
    • LinkingAttributes Module
      • Fully supported
    • ObjectLinking Module
      • Fully supported, but only links in HTML documents can be overridden using the fragment attribute.
  • Transition Effects functionality
    • BasicTransitions Module
      • Subtypes are recognized but not implemented, the default subtype for the transition type is always used.
      • Type miscShapeWipe is recognized but not implemented.
      • Timing for most of the iris wipes is incorrect.
      • The direction attribute is recognized but ignored (direction is always forward).
      • Specifying multiple transitions in the transIn and transOut attributes is not supported.
      • When both the from and the to source of a transition are videos the from source will usually be frozen.
    • CoordinatedTransitions Module
      • Fully supported
    • TransitionModifiers Module
      • The TransitionModifiers attributes are recognized but ignored.
  • Media functionality
    • BasicMedia Module
      • Fully supported
    • MediaClipping Module
      • Fully supported
    • MediaParam Module
      • Not yet supported
    • BrushMedia Module
      • Fully supported
    • MediaAccessibility Module
      • Fully supported in that the readIndex attribute is parsed correctly. There is no support for actually reading out the alt, title, and longdesc attributes.
