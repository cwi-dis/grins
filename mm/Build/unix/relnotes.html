GRiNS for Unix Release Notes

These are the release notes for the GRiNS editor and player Release 1.0beta for Silicon Graphics IRIX 6.5 and for Sun Solaris 2.5. More information on GRiNS can be obtained at http://www.oratrix.com/GRiNS, or by email to grins-request@oratrix.com.

What (sparse) documentation on GRiNS is available at this time can also be found on the homepage.

Prerequisites

The SGI version of the GRiNS editor and player have only been tested on an SGI O2 running IRIX 6.5.2. Other systems may or may not work.

The GRiNS editor and player use a number of system shared libraries. Not all of these libraries are installed by default. Ask your system administrator to install these images if they haven't been installed already. Which images are installed can be seen with the standard program versions. Shared libraries from the following images are used:

  • compiler_eoe.sw.lib
  • dmedia_eoe.sw.audio
  • dmedia_eoe.sw.lib
  • eoe.sw.base
  • eoe.sw.gfx
  • motif_eoe.sw.eoe
  • x_eoe.sw.eoe

Installation instructions

For IRIX-6.5 (and later) on SGI systems, you have two options when installing the GRiNS Editor and Player: using the SGI inst tools or using the gzipped Tar distribution. For Sun/Solaris the only option is usgin a gzipped Tar distribution.

  1. Using grins.tardist or grinsed.tardist: Download the file (grins.tardist for the Player-only or grinsed.tardist for the Editor and Player). If your browser is setup correctly, it will start the SGI Software Manager automatically. If not, you can extract the tardist file in a new directory and start the Software Manager (swmgr) or inst manually. Point either program to the directory where the extracted files reside to start the installation process. From the Software Manager or inst, select the subsystems you want to install (in the player-pnly version, the product is called grins instead of grinsed and the editor subsystem is not present):

    • grinsed.sw.player contains the GRiNS Player and is required.
    • grinsed.sw.editor contains the GRiNS Editor and is required.
    • grinsed.sw.templates contains editing templates for the GRiNS Editor and is required
    • grinsed.help contains help information and is recommended.
    • grinsed.relnotes contains the release notes and is also recommended.

    Important Note: You need superuser (root) privileges to use the inst installation method. The GRiNS Player and Editor (and the supporting files) will be installed in /usr/local/grins.

  2. Using grins.tar.gz or grinsed.tar.gz: If you don't have super-user privileges, download the appropriate file (grins.tar.gz for the Player-only version of GRiNS and grinsed.tar.gz for the Editor and Player) and extract it in a directory of your choice. The command to do this is something like this: gzip -cd grins.tar.gz | tar xf - (for the GRiNS Player), or gzip -cd grinsed.tar.gz | tar xf - (for the GRiNS Editor). The tar file contains the same files as the tardist files and in the same relative configuration.

Platform dependencies in media handling

There are some differences between the various platforms. See the chart of supported media types on the GRiNS website for the supported media types on the various supported platforms.

Access methods

The GRiNS player supports the following access methods (URL schemes).

  • file
  • http
  • ftp
  • data
  • gopher

Unsupported SMIL features

There are only a few SMIL features that are not supported by the GRiNS player. Documents that make use of these features are parsed correctly, but the features are ignored.

  • begin and end attributes in the anchor element.
  • fit="fill" and fit="scroll" attribute values in region element.
  • name="pics-label" attribute value in meta element.
  • alt attribute in media object elements.
  • fill attribute.
