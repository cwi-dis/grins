GRiNS Player for Windows Release Notes

These are the release notes for the GRiNS player for Windows 95/98/NT4, Release 0.2, patchlevel 1. More information on GRiNS can be obtained at http://www.cwi.nl/GRiNS, or by email to grins-request@cwi.nl.

What (sparse) documentation on GRiNS is available at this time can also be found on the homepage.

Prerequisites

For decent performance you probably need at least a 100Mhz Pentium with 16Mb, but 32Mb is definitely preferred. You need about 15Mb of free diskspace.

You need Python 1.5 with PythonWin and preferrably Microsoft Internet Explorer 3.02 or later (see below for details) . All of these are included in the distribution, see below.

For some older versions of Windows 95, and possibly NT too, you may need a COM patch and/or an Active Movie patch. See the problems section below for the symptoms and the solution.

Installation instructions

The order in which you should install things is to first install Python and Internet Explorer (if needed) and only then the GRiNS player. You can skip either of the first steps if you already have that package installed, but do note that GRiNS needs exactly version 1.5 of Python, and a specific build number at that.

If you have already installed Python it may be prudent not to install this version of GRiNS but wait for a self-contained version, which should be available in August.

It is possible to use the GRiNS player without installing Internet Explorer, see the section on Internet Explorer dependency below.

Install everything in the following order:

  • Python 1.5 and PythonWin (the order is important):
    1. Run python15.exe to install the Python 1.5 core.
    2. Run win32-104.exe.
    3. Run win32com-103.exe.
    4. Run pywin-107.exe. These last three steps install the PythonWin extensions.
  • To install Internet Explorer run msie302r.exe.
  • Install the GRiNS player by running grins.exe.

Once the installation is complete, you may run the viewer using the start menu, the desktop shortcut or by double-clicking a .smi or .smil file.

Issues resolved

The following issues have been fixed since the previous release:

  • The "open file" dialog works again.
  • The .smi/.smil filetype issues have been solved.

Customization

One aspect of GRiNS can be customized: the handling of the test attributes such as bitrates and language (see section 4.4 of the SMIL specification). See the examples section on the homepage for details.

Non-technical feedback should be sent to grins-request@cwi.nl.

The SMIL language

The GRiNS player supports the complete SMIL 1.0 definition except where noted below. A conforming SMIL document can be correctly parsed, but a few features are not implemented.

Platform dependencies in media handling

There are some differences between the various platforms. See the chart of supported media types on the GRiNS website for the supported media types on the various supported platforms.

Access methods

The GRiNS player supports the following access methods (URL schemes).

  • file
  • http
  • ftp
  • data
  • gopher

Unsupported SMIL features

There are only a few SMIL features that are not supported by the GRiNS player. Documents that make use of these features are parsed correctly, but the features are ignored.

  • begin and end attributes in the anchor element.
  • fit="fill" and fit="scroll" attribute values in region element.
  • name="pics-label" attribute value in meta element.
  • alt attribute in media object elements.
  • fill attribute.

Compatability problems

If you are running Windows 95 or NT 3.51 and do not have a recent release of Internet Explorer, you may encounter the following error when starting Pythonwin: 


	import win32api [or some other win32 module]
	ImportError: DLL load failed: One of the library files needed to run this application 
	cannot be found.

In that case run oadist.exe to update COM.

Sound and video problems

If sounds or videos are not played correctly on your machine (e.g. *.aiff or *.mpeg) try installing Active Movie by running amovie.exe. For details see grins-mci.txt.

Internet Explorer dependency

MS Internet explorer is needed in order to use MSIE functionality in html pages or just components using the HTML channel. If you do not want to depend on MSIE, follow the following steps.

In the PyDLLs directory you will find the file Htmlex.pyd, HtmlexMS.pyd and HtmlexW.pyd. The two first are the same. The working file is always Htmlex.pyd. If you want to use the Webster HTML Control, which is totally independent from MSIE, the file you need is HtmlexW.pyd. You just need to rename it to Htmlex.pyd and get rid of the old file. By that moment the working pyd file will be the Webster version and not MSIE version. The MSIE version will be at HtmlexMS.pyd. If you want to switch back to MSIE control you should follow the same steps, keeping a copy of Htmlex.pyd (W) renaming it to HtmlexW.pyd and renaming HtmlexMS.pyd to Htmlex.pyd.

In the next version the choice of the HTML control will be made through the setup process. But since it was a change at the last moment, we thought we should release this version, even if it is not done automatically through the setup process.

We suggest the following: If you have MSIE installed, use the default (MS) version. If you have no MS installed, use the "Webster version", although it lacks high-level functionality such as Java. The choice is yours.
