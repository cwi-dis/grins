Webster Pro Release Notes


  Webster Pro V2.7 Release Notes
Previous release notes are available for Webster Pro v2.6.

The 32-bit Webster Pro Control was built with Visual C++ 4.2 and can be used with any matched set of MFC DLLs dated 2-August-96 or later. It has been tested with the MFC DLLs included in the installation kit (which are from VC++ 4.2b) as well as with the sets from VC++ 5.0 and VC++ 6.0. For more information see the Help file section entitled "Redistributing the Webster Pro Control" or the redistribution notes page.

Note: If you are switching from a demo or beta build to the release version of the control in a forms-based application (VB, MS-VC++, Borland C++ Builder etc.), you should delete the demo Webster Pro from the form and then insert the release version in it's place so that the license information is updated. Keep the control ID the same (e.g. WebsterPro1) and you will not have to change any code.

New Properties

  • The new ActiveColor property determines the highlight color for active (pressed) hyperlinks, unless overridden by the specification of an ALINK attribute in a BODY tag. To disable this feature, set the ActiveColor property to the same value as the AnchorColor property.

  • The new HoverColor property specifies the color with which to highlight a hyperlink when the mouse pointer hovers over it. To disable this feature, set the HoverColor property to the same value as the AnchorColor property.

  • The new AnchorUnderline property determines how hyperlink text is rendered. This property can be set on the Misc property page. Possible value are:

    0 AnchorUnderlineNever Never underline hyperlinks.
    1 AnchorUnderlineAlways Always underline hyperlinks (the default).
    2 AnchorUnderlineOnHover Underline hyperlinks on mouse hover.

  • The new StatusDisplay property enables or disables the automatic display of a status page when a page load fails. It is exposed on the Misc property page.

    Set to StatusDisplayPage (=1, the default) to display the auto-status page for failed loads as in previous versions. This status "placeholder" page stays in cache until aged out or dismissed.

    Set to StatusDisplayBarOnly (= 0) to disable the atomatic generation of a status page upon load failure. In this case, the only UI is the status on the status bar, so you may want to provide your own failure indication for the user (e.g., a dialog box). Note also that a status "placeholder" page is not kept in cache for failed loads in this case.

  • The new SocketTimeout property sets the socket timeout in seconds. It is exposed on the Misc property page.


Bug Fixes

  • Avoid vertically breaking text lines when printing.

  • Eliminated a thread race that caused double loads of images when loaded via deferred pure-image (dummy page) loads. This happened when the content-type of a page HTTP header specified an image but this was not initially discernable (by extension) in the original URL.

  • VB-specific fix: Tab and arrow key presses in HTML Forms fields should no longer result in focus shifting to another control on the VB form. When the Tab key is pressed in an HTML Form control the focus shifts to the next HTML Form control.

  • Improved initial jump-to-internal-anchor in subFrames.


Other changes

  • The new Pages property page has been split off from the Misc property page to accomodate new properties.

  • The ALINK attribute of the BODY tag is now supported and determines the color of active (pressed) hyperlink highlighting.

  • The WRAP attribute of the TEXTAREA tag is now supported. TEXTAREA fields now word-wrap by default, unless WRAP=OFF is specified.

  • In Table cells, text-element tags that don't have matching end tags are now automatically closed at the end of the cell. This means that fonts, colors and other text attributes specified inside a Table cell do not affect text outside the cell in which they are specified even if not explicitly closed by the HTML. This matches the current behavior of other browsers.

  • When an HTML Form is newly made visible, focus is now set to the first Form control only if the OCX currently owns the focus.

  • The page title is exposed via the PageTitle property and the GetTitle method as soon as it first becomes available.

  • Improved scroolbar automatic sizing behavior after layout to reduce flicker.

  • For compatibility with other browsers: don't form-decode URLs.
