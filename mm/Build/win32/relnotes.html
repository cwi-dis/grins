GRiNS Editor for Windows Release Notes

These are the release notes for the GRiNS editor and player for Windows 95/98/NT4, Release 0.5, patchlevel 6. More information on GRiNS can be obtained at http://www.oratrix.com/GRiNS, or by email to grins-request@oratrix.com.

What (sparse) documentation on GRiNS is available at this time can also be found on the homepage.

Prerequisites

For decent performance you probably need at least a 100Mhz Pentium with 32Mb, but more memory and a faster processor are definitely preferred. You need about 10Mb of free diskspace.

For some older versions of Windows 95, and possibly NT too, you may need a COM patch and/or an Active Movie patch. See the problems section below for the symptoms and the solution.

Installation instructions

Just execute the installer and follow the directions of the wizard.

Feedback

Bug reports and other technical feedback should be sent to grins-support@oratrix.com, please check the feedback section on the homepage for details.

Non-technical feedback should be sent to info@oratrix.com.

Example documents

A number of example documents can be found in the Examples directory. More examples will be added to the GRiNS home page over time.

License keys

To start the editor you need a license key. You may have been provided with a temporary license key in the same message in which you received the password for the download area. If not you can send an empty mail message to evalutation-license@oratrix.com and we will send you a time-limited key, or we will mail you details on how to proceed.

Platform dependencies in media handling

There are some differences between the various platforms. See the chart of supported media types on the GRiNS website for the supported media types on the various supported platforms.

Access methods

The GRiNS player supports the following access methods (URL schemes).

  • file
  • http
  • ftp
  • data
  • gopher

Unsupported SMIL features

There are only a few SMIL features that are not supported by the GRiNS player. Documents that make use of these features are parsed correctly, but the features are ignored.

  • begin and end attributes in the anchor element.
  • fit="fill" and fit="scroll" attribute values in region element.
  • name="pics-label" attribute value in meta element.
  • alt attribute in media object elements.
  • fill attribute.

Windows user interface open issues

GRiNS can render HTML either through the Internet Explorer HTML control (part of IE4) or the built-in WebsterPro HTML control. This is user-selectable throught the Preferences dialog. In some circumstances the IE control causes the player/editor to crash. If this happens, use the WebsterPro control.

Each control has its good and bad points. IE4 has more features but always shows scrollbars which is not very pretty. Webster is a HTML 3 control only, and has problems with some things like using tags in URLs and accessing files via network neighborhood naming. Both controls can be slow rendering the data on the screen.

If sounds or videos are not played correctly on your machine (e.g. *.aiff or *.mpeg) try installing Active Movie, also sometimes known as Media Player. It is included in Service Pack 4 for NT, and for Windows 95 you should see Microsoft's Media Player download page for details.

Platform-specific issues

There are also a number of features that are not supported on all platforms, or not supported identically.

As issues are reported they are added to the release notes and open issues sections on the GRiNS home page.
