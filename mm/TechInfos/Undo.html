UNDO/REDO

(Sjoerd and Alain. Last update: 21 February 2001)

 

1) design issues

Test of differents softwares

Word 2000 behavior

for each operation:
- stack the operation into the undo list
- clear the redo list

for each undo operation:
- unstack the operation from the undo list, and apply the reverse operation
- stack the operation into the redo list

for each redo operation:
- unstack the operation from the redo list, and apply the reverse operation
- stack the operation into the undo list

notes:

- keep undo and redo into two separates menu entries
- all dialog box properties are modal, and contains (Ok/Cancel button). It avoids consistent pb with undo/redo.
- you can't undo general preference
- granularity level: word (not character), ...

Adobe photoshop behavior

- manage undo/redo, but keep only one level
- keep only one menu entry (swap between undo/redo)

Emacs

- keep only one menu entry (undo), but behaves like there was undo and redo
- manage multi-levels

Sugestion

The way that Word 2000 behaves seems clear and poweful enough. In addition, it's the way of the current GRiNS implementation.
So, we suggest to keep this behavior

warnings

- we have to just check if there is not consistence pb with UNDO/REDO on dialog boxes
- to evaluate the size of undo entries just to know if we can manage a unlimited number of undo
